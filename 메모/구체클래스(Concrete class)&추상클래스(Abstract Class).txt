Concrete Class(구상, 구현, 구체 클래스) 

1. 구체적인 클래스-> 인스턴스화 가능
2. 일반적인 클래스
3. 인스턴스를 만들수 있는 클래스
4. 추상메소드를 가질수없음 -> 항상 메소드의 내부가 구현 되어있어야함 -> 순수가상함수 못씀
5. 추상클래스가 아닌 모든 클래스


Abstract Class(추상클래스)-> 순수가상함수

1. 추상적인 클래스, 인스턴스화 불가능
2. 인스턴스를 만들수 없는 클래스-> 순수가상함수가 포함되어 있으면 직접 객체를 만들수없다
3. 클래스 앞에 abstract를 붙여 인스턴스화를 금지
4. 구체 메소드, 추상 메소드(순수가상함수) 둘다 가질수있음
5. 추상클래스에서 파생된 구체클래스는 추상클래스의 메소드들을 모두 구현해야됨-> 추상클래스에 선언된 순수가상함수를 모두 구현해야함
